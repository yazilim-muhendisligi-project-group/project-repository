CREDENTIALS GUIDE - Bahar Kiraathanesi
=======================================

Bu belge, proje temizligi sirasinda degistirilen dosyalari ve
yerel ortamda sifrelerin nasil ayarlanacagini aciklamaktadir.

DEGISTIRILEN DOSYALAR:
----------------------

1. src/main/resources/db/setup_database.sql
   - 'yonetici' kullanicisinin sifresi 'YOUR_PASSWORD_HERE' ile degistirildi
   - 'admin' kullanicisinin sifresi 'YOUR_PASSWORD_HERE' ile degistirildi

2. src/main/resources/db/update_products.sql
   - 'yonetici' kullanicisinin sifresi 'YOUR_PASSWORD_HERE' ile degistirildi

YEREL ORTAMDA SIFRELERI AYARLAMA:
---------------------------------

1. Yukaridaki SQL dosyalarini acin.

2. 'YOUR_PASSWORD_HERE' ifadelerini gercek sifrelerinizle degistirin.

3. Ornek:
   ONCESI:  VALUES ('yonetici', 'YOUR_PASSWORD_HERE', 'admin')
   SONRASI: VALUES ('yonetici', 'gercek_sifreniz', 'admin')

4. Degisiklikleri kaydedin ancak GIT'E COMMIT ETMEYIN.

5. SQL scriptlerini MySQL veritabaninizda calistirin.

ONEMLI NOTLAR:
--------------

- Gercek sifreleri ASLA git repository'sine eklemeyin.
- Yerel degisikliklerinizi .gitignore ile koruyabilirsiniz.
- Hassas bilgiler icin ortam degiskenleri (environment variables) kullanmayi dusunun.
- Bu dosya gecici bir rehberdir ve ileride silinebilir.
