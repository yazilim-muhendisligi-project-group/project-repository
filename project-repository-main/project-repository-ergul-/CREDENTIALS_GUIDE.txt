CREDENTIALS GUIDE - Bahar Kiraathanesi
=======================================

Bu belge, proje temizligi sirasinda degistirilen dosyalari ve
yerel ortamda sifrelerin nasil ayarlanacagini aciklamaktadir.

DEGISTIRILEN DOSYALAR:
----------------------

1. src/main/resources/db/setup_database.sql
   - 'yonetici' kullanicisinin sifresi 'YOUR_PASSWORD_HERE' ile degistirildi
   - 'admin' kullanicisinin sifresi 'YOUR_PASSWORD_HERE' ile degistirildi

2. src/main/resources/db/update_products.sql
   - 'yonetici' kullanicisinin sifresi 'YOUR_PASSWORD_HERE' ile degistirildi

3. src/main/java/com/baharkiraathanesi/kiraathane/database/DatabaseConnection.java
   - MySQL veritabani sifresi kaldirildi (bos string yapildi)
   - Sifre artik ORTAM DEGISKENI (environment variable) ile alinmaktadir

VERITABANI BAGLANTISI - ORTAM DEGISKENLERI:
-------------------------------------------

DatabaseConnection.java dosyasi su ortam degiskenlerini kullanir:

  DB_HOST     -> Varsayilan: localhost
  DB_PORT     -> Varsayilan: 3306
  DB_NAME     -> Varsayilan: bahar_db
  DB_USER     -> Varsayilan: root
  DB_PASSWORD -> Varsayilan: (bos)

ORTAM DEGISKENLERINI AYARLAMA:
------------------------------

YONTEM 1 - Terminal'de gecici olarak (macOS/Linux):
  export DB_PASSWORD="gercek_sifreniz"
  export DB_USER="kullanici_adiniz"

YONTEM 2 - ~/.zshrc veya ~/.bash_profile'a kalici olarak ekleyin:
  echo 'export DB_PASSWORD="gercek_sifreniz"' >> ~/.zshrc
  source ~/.zshrc

YONTEM 3 - IntelliJ IDEA'da:
  1. Run > Edit Configurations
  2. Uygulamanizi secin
  3. "Environment variables" alanina ekleyin:
     DB_PASSWORD=gercek_sifreniz;DB_USER=root

SQL DOSYALARINI KULLANMA:
-------------------------

SQL dosyalari src/main/resources/db/ klasorune tasindi.
Veritabanini kurmak icin:

1. SQL dosyasini acin
2. 'YOUR_PASSWORD_HERE' ifadelerini gercek sifrelerinizle degistirin
3. MySQL'de calistirin:
   mysql -u root -p < src/main/resources/db/setup_database.sql
